# AGENTS.md

## 构建、检查和测试命令

### 构建命令

要构建项目，请使用以下命令：

```bash
vite build
```

### 检查命令

要检查代码库，请运行：

```bash
eslint .
```

### 测试命令

要运行测试，请使用：

```bash
npm test
```

要运行单个测试，您可以指定测试文件：

```bash
npm test -- path/to/test/file.spec.ts
```

## 代码风格指南

### 导入

- 使用命名导入特定组件和函数。
- 逻辑地分组导入：
  - 外部库
  - 内部模块

### 格式化

- 遵循 ESLint 配置中定义的 Prettier 规则。
- 使用 2 个空格进行缩进。

### 类型

- 对所有变量和函数参数使用 TypeScript 类型。

### 命名约定

- 对变量和函数使用 camelCase。
- 对组件名称使用 PascalCase。

### 错误处理

- 对异步操作使用 try-catch 块。
- 适当地记录错误并提供用户友好的消息。
